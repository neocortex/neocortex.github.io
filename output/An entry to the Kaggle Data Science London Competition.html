Kaggle: Data Science London¶

In this notebook I am going to show how to build a simple model for the Kaggle Data Science London competition using scikit-learn.

The data set at hand is synthetic data that consists of 40 features, representing objects from two classes (labeled as 0 or 1). Hence, the present machine learning task is a binary classification problem. The training set has 1000 samples and the testing set has 9000. Because the data is synthetic, the features do not represent anything meaningful or interpretable.

First, let's import some libraries that we will be using along the way:

  1. The last 3 components do not provide information. 100% of the variance is explained by using the first 37 principal components.
  2. Most of the variance (> 80%) is explained by the first 12 pricipal components.

Using Support Vector Machines¶

Ok, let's jump straight into training a classifier. There are many methods to try out here, including Logistic Regression, LDA, Random Forests, etc. Another straight-forward choice (which I make here) are Support Vector Machines (SVMs). In general, the first step I would take after deciding on a method, is to perform model selection by tuning the crucial hyperparameters of the model. In particular, SVMs have two hyperparameters that need to be tuned: C and gamma. You could also try out different kernels, but here I will stick simply to the default RBF kernel and do a gridsearch over C and gamma. In addition, I will make use of the scikit-learn Pipeline object, because I intend to first scale the data to zero mean and unit variance in each cross-validation fold.

In [7]:
pip = Pipeline([('scaler', StandardScaler()), ('clf', SVC(kernel='rbf'))])
C_range = np.logspace(-2, 3, 20)
gamma_range = np.concatenate((np.asarray([0]), np.logspace(-8, 0, 20)))
params = {'clf__C': C_range,
          'clf__gamma': gamma_range}
gs = GridSearchCV(pip, params, cv=5, n_jobs=-1)
gs.fit(X_train, y_train);

Now that the gridsearch over the specified parameter range is completed, we can inspect the best validation score and the best estimator:

To get a better overview over the performace of the different models, I define a helper function that plots the validation score for each pair of hyperparameter values:

In [9]:
def plot_grid(grid_search):
    """ Plot the validation score for each value of C and gamma from a gridsearch on an SVM.
        
        :param grid_search: A scikit-learn GridSearchCV object.
    
    """
    grid_scores = grid_search.grid_scores_
    gammas, Cs, scores = [], [], []
    gamma_str, c_str = ('clf__gamma', 'clf__C') if isinstance(
        gs.best_estimator_, Pipeline) else ('gamma', 'C')
    for grid in grid_scores:
        gammas.append(grid.parameters[gamma_str])
        Cs.append(grid.parameters[c_str])
        scores.append(grid.mean_validation_score)
    gammas = np.asarray(gammas)
    Cs = np.asarray(Cs)
    scores = np.asarray(scores)
    scores = scores.reshape(len(C_range), len(gamma_range))
    pl.figure(figsize=(10,10))
    pl.imshow(scores, interpolation='nearest', cmap=pl.cm.spectral)
    pl.xlabel('gamma')
    pl.ylabel('C')
    pl.xticks(np.arange(len(gamma_range)), gamma_range, rotation=45)
    pl.yticks(np.arange(len(C_range)), C_range)
    pl.colorbar()

Using Principal Components¶

Let's test whether can improve our model by using principal components as features. Therefor, we use the PCA as a first step in a scikit-learn Pipeline object, and perform a gridsearch over all the possible numbers of principal components, as well as over reasonable ranges of gamma and C:

In [21]:
pca = PCA(whiten=True)
pip = Pipeline([('pca', pca), ('clf', SVC(kernel='rbf'))])
params = {'pca__n_components': np.arange(2, 40),
          'clf__gamma': np.linspace(0, 0.02, 10),
          'clf__C': np.linspace(0.9, 3, 20)}
gs = GridSearchCV(pip, params, cv=5, n_jobs=-1)
gs.fit(X_train, y_train);

Predicting¶

Now that we have a decent classifier, we can predict the labels of the test samples, and save these in a CSV-file that we can upload to the competitions' page on Kaggle to see how we perform on the leaderboard (spoiler: we reach 0.94188).

In [23]:
def save_predictions(preds):
    ids = np.arange(1, 9001)
    with open('predictions.csv', 'w') as fp:
        a = csv.writer(fp, delimiter=',')
        header = ['Id', 'Solution'],
        a.writerows(header)
        a.writerows(zip(ids, preds))
In [24]:

Using Gaussian Mixture Model¶

The following part is not my own idea, but merely something that has been discussed in the competitions' forum, and thus I wanted to try it out here. If you plot the distributions of the principal components, it seems like they all are (except the las three components) Gaussian distributed. Therefore, it has been proposed to use a Gaussian Mixture Model (GMM) on the principal components and feed the predicted probabilities into a Support Vector Machine as features for classification. I am not going to perform a detailed gridsearch over the parameters of the GMM, but simply use what has been reported to work well (n_components = 4, covariance_type = 'full').

In [25]:
from sklearn.mixture import GMM
In [26]:
pca = PCA(n_components=37, whiten=True)
X_pca = pca.fit_transform(X_train)
gmm = GMM(n_components=4, covariance_type='full')
gmm.fit(X_pca)
probs = gmm.predict_proba(X_pca)
In [27]:
cross_val_score(SVC(C=3.), probs, y_train, cv=5, n_jobs=-1).mean()
Out[27]:
0.99299999999999999

Indeed, using the GMM we reach a validation score as high as 99%! Hence, we can train our SVM on the probabilities of the GMM, and predict the test set samples using the same steps as during training:

  1. PCA
  2. GMM
  3. SVM
In [28]:
clf.fit(probs, y_train)
X_pca = pca.transform(X_test)
probs = gmm.predict_proba(X_pca)
preds = clf.predict(probs)
save_predictions(clf.predict(probs))
